- Jira is still the operational source; admin adds machine metadata, repair costs,

Current Cost State
- Costs first view now has:
  category filter
  subcategory filter
  date range filter
  text search
- Cost search matches:
  Jira key
  category
  subcategory
  summary
  description
  creator
  reporter fallback
- Search bar is full-width and shown above the other cost filters.
- Selected-period ticket list in the costs view is now read-only and uses the same
  list-card formatting/spacing as the main page tickets list.
- Inline cost editing was intentionally removed from that selected-period list.
- Manual machine-cost management still exists in the costs page when a machine is selected.
- Costs view now defaults to All tickets on first load/reset, not Last 7 days.
- Last 7 days now means:
  today included
  previous 6 days included
  local date boundaries, not UTC-shifted boundaries

Current Ticket Modal Cost State
- Ticket modal header now shows:
  key
  summary
  created
  updated
  time spent
  status
- Priority was intentionally removed from the modal header because it is not useful
  in the cost flow.
- Cost area now centers on one clearer ticket-cost workflow:
  direct ticket cost
  optional additional asset costs
- Direct ticket cost stays visible by default.
- Additional asset costs only appear when:
  the checkbox is enabled
  or existing extra asset-cost rows already exist for that ticket/machine
- Modal summary cards now show:
  ticket time spent
  direct ticket cost
  total cost
- Current behavior of total cost in the modal:
  total = direct ticket cost + additional asset costs

Important Cost Model Note
- Current storage is still split:
  ticket direct cost = /api/admin/ticket-costs
  asset-level extra costs = /api/admin/manual-entries
- UI is clearer now, but the underlying model is still mixed.
- Long term desired direction:
  ticket-linked repair costs should live with the Jira ticket
  non-ticket asset costs should live on the asset/costs page, not inside the ticket modal

Current Maintenance State
- Planned maintenance is now local-first in the admin panel.
- Main maintenance files:
  app/admin/components/MaintenanceSection.tsx
  app/admin/hooks/useAdminMaintenance.ts
  app/api/admin/planned-maintenance/route.ts
  app/api/admin/planned-maintenance/[id]/route.ts
- Current maintenance behavior now includes:
  create in local DB only
  create works with or without notification recipients selected
  edit in UI + local DB
  complete/reopen in local DB
  delete from local DB
  legacy Jira-linked item handling is still guarded in edit/delete paths if old linked rows still exist
  stale linked-row repair still exists on GET for old Jira-linked maintenance rows
- Planned maintenance modal now uses:
  date-only event selection
  optional scheduled hours
  start time only or start + end time
- Scheduled hours are shown in:
  calendar event card
  maintenance notification emails
- Visual state:
  maintenance calendar cards now use a neutral green project-style background
  status-specific dots / legends / colored status cues were intentionally removed
- Current CRUD sanity check status:
  create / edit / complete / delete paths were reviewed after the local-only maintenance change
  no current blocking CRUD issue was found after the no-recipient create fix
  current automated coverage is still light and mostly component-level, not full route integration coverage
- Notifications use Resend.
- Shared mailer:
  lib/plannedMaintenanceMailer.ts
- Test sender route intentionally kept:
  app/api/send/route.ts

Current Auth/Security Notes
- Login wording is email-only now.
- Bootstrap admin login is setup-only and gated by:
  ALLOW_ADMIN_BOOTSTRAP=true
  zero existing users
- Mutating admin APIs already enforce admin auth + same-origin checks.

Current Inventory / Asset Detail State
- Inventory first view is read-only by design.
- Model, serial number, and manufacturer are no longer editable in the inventory list.
- Inventory first view now has:
  full-width search bar
  category filter
  subcategory filter
  refresh action aligned bottom-right on desktop
- Inventory row details are intentionally slightly muted so they read as view-only.
- Open asset details link was toned down visually so it does not overpower the row.
- Asset detail page now owns machine detail editing.
- On asset detail page:
  machine key stays read-only
  model / serial number / manufacturer can be edited inline under the title
  editing starts only after pressing Edit details
  edit state should keep the same layout and typography as read-only state
  main active-state visual cue is white background on editable fields
